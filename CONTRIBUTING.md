# Contributing Guidelines

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.

## How to Contribute
1. Fork the repository.
2. Create a new branch for your change:
   - `git checkout -b feature/your-feature-name`
3. Make your updates and commit with a clear message:
   - `git commit -m "Describe your change"`
4. Push your branch to your fork:
   - `git push -u origin feature/your-feature-name`
5. Open a Pull Request (PR) describing what you changed and why.

## Reporting Bugs
If you find a bug, please open an Issue and include:
- A short summary of the problem
- Steps to reproduce
- Expected vs. actual behavior
- Screenshots/logs if available

## Suggestions and Enhancements
New ideas are encouraged. Open an Issue explaining:
- The improvement you’re proposing
- Why it’s useful
- Any examples or references that help
